## 新增需求

### 需求:Agent 会话内工具调用去重

Agent 执行器必须在单次会话内维护工具调用结果缓存。当 LLM 在同一会话中以相同参数再次调用同一工具时，系统必须直接返回首次调用的结果，禁止重复执行。

#### 场景:相同参数重复调用直接返回缓存

- **当** LLM 在同一会话中第一次调用 `get_daily_history(stock_code="600519", days=60)` 并成功返回
- **且** LLM 在同一会话中再次以相同参数调用同一工具
- **那么** 系统必须直接返回第一次调用的缓存结果
- **且** 禁止再次调用外部 API

#### 场景:不同参数调用正常执行

- **当** LLM 在同一会话中先后调用 `get_daily_history(stock_code="600519", days=60)` 和 `get_daily_history(stock_code="600519", days=365)`
- **那么** 系统必须分别执行两次调用
- **且** 第二次调用不受第一次缓存影响

#### 场景:会话结束后缓存清理

- **当** Agent 会话结束（完成或超时）
- **那么** 该会话的工具调用缓存必须被释放
- **且** 下一会话不受前一会话缓存影响

### 需求:筹码分布查询感知熔断状态

`get_chip_distribution` 工具必须在调用前检查 Akshare 筹码数据源的熔断器状态。如果熔断器处于断开（open）状态，必须直接返回跳过信息，禁止尝试调用外部 API。

#### 场景:熔断器断开时跳过调用

- **当** Agent 调用 `get_chip_distribution(stock_code="600519")`
- **且** Akshare 筹码数据源的熔断器处于断开状态
- **那么** 系统必须直接返回 `{"status": "skipped", "reason": "circuit_breaker_open"}`
- **且** 禁止尝试连接外部 API
