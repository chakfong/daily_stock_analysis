## 新增需求

### 需求:K线加载优先使用本地数据库并增量补全

`load_history_df` 函数必须在调用外部数据源之前优先查询本地数据库。当数据库中已有该股票的部分历史数据时，系统禁止全量重新拉取，必须仅拉取缺失的日期范围（增量补全）。

#### 场景:数据库有足够数据时直接返回

- **当** 数据库中存在该股票至少 30 条 K 线记录，且最新日期距目标日期不超过 3 个交易日
- **那么** 系统必须返回 `("db_cache", df)`，不调用任何外部 API
- **且** 返回的 DataFrame 必须包含请求天数范围内的全部数据

#### 场景:数据库有数据但有缺口时增量补全

- **当** 数据库中已有该股票的历史 K 线，但最新日期距目标日期超过 3 个交易日
- **那么** 系统必须计算缺口日期范围（DB 最新日期+1 天 到 目标日期）
- **且** 仅对缺口范围调用 DataFetcherManager 获取数据
- **且** 将新获取的数据追加写入数据库
- **且** 返回合并后的完整 DataFrame

#### 场景:数据库无数据时全量拉取

- **当** 数据库中不存在该股票的任何 K 线记录
- **那么** 系统必须调用 DataFetcherManager 获取完整数据
- **且** 将获取的数据写入数据库
- **且** 行为与优化前保持一致

#### 场景:不同窗口大小的请求复用缓存

- **当** Agent 先请求 `days=60` 的 K 线，随后又请求 `days=365` 的 K 线
- **那么** 第二次请求必须复用第一次已缓存的近期 60 天数据
- **且** 仅对缺失的更早期数据（第 61-365 天）调用外部 API
