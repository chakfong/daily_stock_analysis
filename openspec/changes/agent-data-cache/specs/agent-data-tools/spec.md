## 修改需求

### 需求:get_daily_history 缓存命中条件放宽

`load_history_df` 的数据库缓存命中条件必须从"最新日期 >= 目标日期"改为"最新日期距目标日期不超过 3 个交易日"。当缓存部分命中时，网络回退路径必须使用精确的日期范围参数仅拉取缺失部分。

#### 场景:缓存略微过时仍命中

- **当** 数据库中某股票最新 K 线日期为昨日（距今日 1 个交易日）
- **且** 请求目标日期为今日
- **那么** 系统必须使用数据库缓存数据
- **且** 仅补拉从昨日到今天缺失的 K 线（如有交易日）
- **且** 返回合并后的完整数据

#### 场景:缓存严重过时触发全量拉取

- **当** 数据库中某股票最新 K 线日期距目标日期超过 3 个交易日
- **且** 数据库中缓存记录数不足 30 条
- **那么** 系统必须回退到全量拉取模式
- **且** 与优化前的行为保持一致
