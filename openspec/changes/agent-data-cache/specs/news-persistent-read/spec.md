## 新增需求

### 需求:新闻搜索优先从数据库读取已持久化结果

新闻搜索工具（`search_stock_news`、`search_comprehensive_intel`）必须在调用外部搜索 API 之前先查询数据库中已持久化的新闻记录。如果存在时效性可接受的缓存结果，禁止调用外部 API。

#### 场景:数据库有新鲜新闻时直接返回

- **当** Agent 调用 `search_stock_news(stock_code="600519")`
- **且** 数据库中该股票 `latest_news` 维度的新闻记录距今不超过 24 小时
- **那么** 系统必须直接返回数据库中的新闻数据
- **且** 禁止调用 SerpAPI 或任何外部搜索 API

#### 场景:数据库无缓存或缓存过期时调用外部 API

- **当** Agent 调用 `search_stock_news(stock_code="600519")`
- **且** 数据库中不存在该股票的新闻记录，或最新记录距今超过 24 小时
- **那么** 系统必须调用外部搜索 API 获取最新新闻
- **且** 将搜索结果持久化到数据库（保持现有行为）

#### 场景:多维情报搜索的缓存隔离

- **当** Agent 调用 `search_comprehensive_intel` 触发多个维度的搜索
- **那么** 每个维度必须独立检查数据库缓存
- **且** 仅对缓存缺失或过期的维度调用外部 API
- **且** 缓存命中的维度必须与 API 获取的维度合并返回

#### 场景:用户明确要求实时新闻时跳过缓存

- **当** Agent 调用搜索工具时传入 `freshness="realtime"`
- **那么** 系统必须跳过数据库缓存检查
- **且** 直接调用外部搜索 API 获取最新结果
